# 对话轮次卡片默认展开设计

## 目标

所有对话轮次卡片在首次渲染时默认展开，让用户无需额外点击即可看到消息、Agent 活动、公开思考摘要和交付结果。用户仍可点击卡片标题手动收起。

## 设计

`ConversationTurnCard` 继续使用原生 `<details>`，但不再根据轮次的 `queued` 或 `running` 状态决定初始展开状态。组件始终为 `<details>` 提供初始 `open` 属性，并保持非受控交互，因此浏览器会在首次渲染后自行处理用户的展开和收起操作。

不增加 React 状态，不持久化收起偏好，也不改变卡片内容、标题或终态展示规则。

## 测试

更新工作区回归测试，确认卡片始终使用默认 `open`，并确认不再存在按轮次状态计算展开状态的逻辑。现有前端测试、TypeScript 检查和生产构建必须继续通过。
