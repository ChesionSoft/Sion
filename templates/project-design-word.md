# 正式产品需求文档（PRD）Word 模板说明

正式 PRD 是对外交付稿，由批准的导出蓝图与正式正文生成，不是 11 个工作流节点的原始拼接。

## 封面

- 文档标题：正式产品需求文档（PRD）
- 项目名称
- 客户名称
- 编制方
- 版本号
- 生成日期

## 目录

预填章节条目（H1–H3）并挂书签超链接，打开文档即可看到目录，无需手动“更新域”。
页码在 Word / WPS 打开后自动刷新。目录标题本身不占用 Heading 样式，避免自引用。

## 正文章节骨架

正式正文按批准的导出蓝图组织，常用章节如下（不适用的小节直接省略，不编造、不留空）：

- 执行摘要
- 需求背景与建设目标
- 用户与使用场景
- 产品方案
- 关键页面与交互
- 数据与接口概要
- 范围、验收标准与风险
- 附录（可选）

## 内容取舍（硬约束）

- 只能包含来源映射为 `confirmed` 或 `confirmed-summary` 的内容，以及显式选择的 `required-disclosure`。
- `omit` 内容、待确认事项、开放问题、agent 建议、历史结论、过程检查清单、未确认假设一律不得进入正式 PRD 与 Word。
- 表格只用于重复可比较的信息；超过半数单元格超过 120 字的表格视为“文字塞表格”，需改为正文或列表。
- 流程图使用 ```flow 代码块，内容仅为一行 2–6 个以 `->` 分隔的节点，渲染为横向流程图；其它代码块不得进入正式正文。

## 样式

- 中文字体 PingFang SC（`eastAsia` 槽位），西文 Aptos；标题、正文、表头、页眉页脚、列表与流程图标签统一使用该字体。
- 覆盖内置 Heading1–3（含 `outlineLevel`）供目录收录；PrdTitle / PrdBody 用于封面与正文段落。
- 正文前含修订记录表；页眉显示项目名与版本，页脚显示「第 x / y 页」。

## 结构与内容校验

正式 Word 生成后，服务端对 OOXML 包做结构与内容校验（纯 Node，无需 LibreOffice/Poppler）：包可读且不小于 1 KB、ZIP 头合法、含 `word/document.xml` 与 `[Content_Types].xml`、`document.xml` 可被 XML 解析器解析、正文 `w:t` 文本非空、至少一个大纲级标题（Heading 1–3）、正式 PRD 中文内容契约通过。校验通过后 Word 才对外可见；失败则返回 422 并保留校验报告。真实排版（字体、分页、目录页码）需在 Word / WPS 中复核。
