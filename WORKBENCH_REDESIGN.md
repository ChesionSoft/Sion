# Sion 工作台重设计

**状态：待评审，未开始实现**  
**日期：2026-07-16**

## 1. 背景与目标

Sion 已完成从网页应用到本地优先桌面应用的技术迁移，但当前界面仍保留了偏深色、偏开发工具的视觉语言：文字密度高、辅助文字过小，模型连接只能新增不能管理，聊天被挤在右栏，文件池也不能承担资料预览职责。

本次重设计的目标是把 Sion 做成适合长期阅读、对话、整理和交付项目设计文档的桌面工作台。它应当有接近 Codex 的清爽、克制和高信息密度，但不复制 Codex 的视觉品牌。

### 目标

1. 改为高可读性的浅色编辑环境，消除大面积黑色和 9–10px 小字。
2. 让“对话”成为项目工作时的主视觉区域，文件资料成为右侧可预览的上下文。
3. 保留用户选择项目目录的自由，同时允许在设置中指定原生目录选择器的默认起点。
4. 让已有模型连接可查看、编辑、设为默认和删除，同时不泄露 API Key。
5. 删除没有实际用户价值的旧项目迁移入口及其旧浏览器兼容逻辑。

### 非目标

- 不增加浏览器搜索、网页抓取、浏览器自动化或 Playwright。
- 不把项目数据改为云端同步，也不把 API Key 写入 `.sion/`。
- 不改变 Agent 的 `delivery` 校验、revision/CAS 保存与并发保护策略。
- 不在本次引入新模型协议、自动更新或多窗口工作流。

## 2. 已确认的产品决定

| 决定 | 结论 |
|---|---|
| 视觉方向 | 使用“编辑部纸张”浅色主题，而不是现有深色终端风。 |
| 字号 | 正文、对话与表单整体上调；12px 是辅助信息下限。 |
| 项目目录 | 新建项目仍由用户通过系统目录选择器决定。 |
| 默认目录 | 在设置中保存“默认项目目录”，只作为下次目录选择器的初始位置。 |
| 历史迁移 | 没有存量用户，删除迁移入口和相关运行时代码。 |
| 模型连接 | 已有连接可进入详情编辑；密钥不可回显，只能保留、替换或删除。 |
| 工作台主区域 | 中间是对话，右侧是文件预览；交付稿通过中间区域的页签访问。 |

## 3. 体验原则

### 3.1 文档优先，而不是终端优先

用户的主要工作是阅读需求、与 Agent 讨论、确认设计稿和导出文档。界面应像一本有秩序的工作笔记：留白充足、文字稳定、状态克制，而不是以装饰性深色背景和密集技术标识吸引注意。

### 3.2 对话是过程，交付稿是结果

对话负责探索和澄清，Markdown 交付稿负责保存被确认的结论。两者需要切换，但不能彼此竞争空间：默认先进入对话，用户随时切到交付稿审阅或编辑。

### 3.3 资料必须可见、可选、可追溯

“文件池”不再只是左下角几个复选框。右侧资料栏需要同时承担导入、选择为 Agent 上下文、查看提取结果和辨识失败状态四种职责。

### 3.4 默认值帮忙，不替用户做决定

默认项目目录只缩短目录选择路径。每次新建仍弹出系统目录选择器，用户可以进入任意位置；Sion 不会擅自把项目写入某个固定目录。

## 4. 信息架构

### 4.1 启动页

启动页保留三个核心入口：新建项目、最近项目、模型连接；新增一个清晰但低干扰的“设置”入口。

```text
┌─────────────────────────────────────────────────────────────────────┐
│ SION                                                        [设置]   │
│ 本地优先的项目设计文档工作台                                         │
│                                                                     │
│ ┌────────────── 新建项目 ──────────────┐ ┌──── 最近项目 ───────────┐ │
│ │ 项目名称                              │ │ 项目 A                 │ │
│ │ 客户 / 作者                           │ │ 项目 B                 │ │
│ │ 默认位置：~/Documents/Projects        │ │ …                      │ │
│ │ [选择目录并创建]                      │ │                        │ │
│ └──────────────────────────────────────┘ └────────────────────────┘ │
│                                                                     │
│ ┌──────────────────── 模型连接 ───────────────────────────────────┐ │
│ │ OpenAI · 已配置 · 默认                    [+ 添加连接]           │ │
│ │ DeepSeek · 已配置                                               │ │
│ └────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

“选择目录并创建”始终调用系统原生目录选择器。若设置了默认项目目录，选择器从该目录打开；若未设置，则使用操作系统默认位置。

### 4.2 设置

设置以轻量弹层或独立设置页呈现，首版只承载应用级目录偏好，避免变成杂项中心。

| 项目 | 行为 |
|---|---|
| 默认项目目录 | 展示当前目录，点击“更改”打开系统目录选择器。 |
| 取消默认目录 | 清空偏好；不会移动、删除或影响既有项目。 |
| 新建项目提示 | 显示“新项目会在你选择目录内创建 `.sion/`”。 |

应用设置存放在操作系统的应用数据目录，例如 `settings.json`；它不属于任何项目，也不写入 `.sion/`。

### 4.3 项目工作台

工作台固定为三栏，核心变更是将聊天从右侧窄栏移至中间。

```text
┌──────────────┬────────────────────────────────────────┬───────────────┐
│ 设计路径      │ 项目名称 / 当前节点                     │ 资料           │
│ 01 基本信息   │ [对话] [交付稿]          保存 · 导出 DOCX │ [＋ 导入]      │
│ 02 建设目标   ├────────────────────────────────────────┤ 文件列表       │
│ 03 角色权限   │ 会话选择  + 新会话                        │ ☑ 客户需求.pdf │
│ …            ├────────────────────────────────────────┤ ☐ 会议纪要.docx│
│              │                                        ├───────────────┤
│              │            对话消息主区域                │ 客户需求.pdf   │
│              │                                        │ 12 页 · 已提取 │
│              │                                        │                │
│              │                                        │ 提取文本预览   │
│              ├────────────────────────────────────────┤                │
│              │ 输入框                     [发送并运行] │                │
└──────────────┴────────────────────────────────────────┴───────────────┘
```

#### 左栏：设计路径

- 固定显示 12 个节点、当前节点和节点状态。
- 只保留节点导航；移除文件池、任务中心等混合职责。
- 宽度约 220px，可滚动；节点名称最小字号 13px。

#### 中栏：对话 / 交付稿

- 默认页签为“对话”。
- 对话页含会话切换、新建会话、消息流、Agent 运行状态、发送框和“预览修改”。
- “交付稿”页使用当前 Markdown 编辑器，保留 revision、保存、节点状态和自定义规则入口。
- 顶部始终展示项目名、当前节点、保存状态和 DOCX 导出。
- Agent 交付仍先在预览弹层中展示完整修改，确认后才写入交付稿。

#### 右栏：资料与文件预览

- 顶部为文件导入按钮和文件清单；复选框代表“纳入本次 Agent 上下文”。
- 点击文件行时，不改变其是否被选入上下文，只切换下方预览。
- 预览显示原始文件名、类型、大小、提取状态、页数或工作表数（若可用）及提取文本。
- PDF、DOCX、XLSX 首版只预览本地提取文本，不嵌入浏览器 PDF 查看器，也不打开网页。
- 预览文本应有字符上限和“已截断”提示，避免超大附件拖慢 UI。
- 无法提取的文件显示失败原因与原件已保留状态，而不是空白预览。

#### 窄窗口策略

桌面窗口小于约 1080px 时，右栏收为“资料”抽屉；中栏始终保留。左侧节点栏可缩窄但不隐藏节点名称。应用最小宽度调整为适合该布局的值，并在最低宽度下避免文字重叠。

## 5. 视觉与排版系统

### 5.1 方向：编辑部纸张

视觉应是温和的纸面、深墨排版与少量工具色：比纯白更耐看，比黑色终端更适合长文档。背景不使用网格、发光、霓虹渐变或过量阴影。

| Token | 建议值 | 用途 |
|---|---|---|
| `--canvas` | `#F6F6F2` | 页面底色，微暖白。 |
| `--surface` | `#FFFFFF` | 面板、编辑区和浮层。 |
| `--ink` | `#1F2421` | 主要文字。 |
| `--muted` | `#65706A` | 次要文字与说明。 |
| `--line` | `#DCE0D9` | 边框和分割线。 |
| `--accent` | `#147A72` | 当前节点、主要操作、已保存状态。 |
| `--warning` | `#B86A32` | 待确认、取消、风险提示。 |
| `--selection` | `#E3F1EE` | 当前行和轻量选中态。 |

### 5.2 字体与字号

不引入网络字体，优先使用系统中可用的中文字体，以保证离线和跨平台表现。

```css
--font-ui: "PingFang SC", "Microsoft YaHei UI", "Noto Sans CJK SC", sans-serif;
--font-reading: "Songti SC", "STSong", "Noto Serif CJK SC", serif;
--font-mono: ui-monospace, "SFMono-Regular", Consolas, monospace;
```

| 元素 | 目标字号 | 说明 |
|---|---:|---|
| 页面标题 | 34–44px | 阅读型衬线字，仅用于少数页面标题。 |
| 节点标题 | 24–28px | 清晰区分当前工作章节。 |
| 正文 / Markdown 编辑器 | 16px / 1.7 行高 | 主要长文阅读与编辑。 |
| 对话消息 | 15–16px / 1.65 行高 | 保证快速扫读与长回复可读。 |
| 输入控件 | 14px | 不使用微型表单字。 |
| 辅助说明 | 13px | 项目路径、时间、状态说明。 |
| 标签 | 最低 12px | 可用字重和颜色区分，而非缩小字号。 |

### 5.3 交互状态

- 主按钮为实心青绿色，危险操作为文字或描边琥珀色，避免整页出现多个高饱和按钮。
- 焦点状态使用 2px 青绿色外环，所有键盘可达元素可见。
- 切换节点、页签和文件预览只使用 120–180ms 的轻微淡入/位移；不做循环动画。
- 通过颜色之外的文字、图标或边框表达“保存中、已保存、失败、待确认”。

## 6. 模型连接管理

### 6.1 从“新增表单”改为“连接管理器”

启动页的模型区域改为连接列表。点击已有连接打开详情面板，不再只能看到“已配置”。

```text
┌──────────── 模型连接 ────────────┐
│ ● OpenAI                  默认   │
│   Responses · gpt-5 · 已配置      │
│                                  │
│   DeepSeek                        │
│   Chat Completions · 已配置        │
│                                  │
│                    [+ 添加连接]   │
└──────────────────────────────────┘
```

### 6.2 连接详情

| 字段 | 已有连接中的行为 |
|---|---|
| 提供商名称 | 可编辑。 |
| API Base URL | 可编辑；仍使用 Base URL 自动拼接 endpoint。 |
| 协议 | 可编辑：Chat Completions / Responses。 |
| 默认模型 | 可编辑，填写精确模型 ID。 |
| API Key | 永不读取或回显。显示“密钥已保存”；用户可输入新值替换。 |
| 默认提供商 | 通过“设为默认”显式切换。 |
| 删除 | 二次确认；同时删除该提供商的系统凭据。 |

新连接仍必须有 API Key；编辑已有连接时，空的 Key 输入框表示保留原密钥，不是清空。用明确的“替换 API Key”开关或按钮避免歧义。

### 6.3 后端行为

- 现有 `ProviderInput` 复用同一稳定 `id` 即可覆盖元数据；前端编辑时不得重新生成 id。
- 新增 `provider_set_default(provider_id)` 命令，原子更新提供商元数据；不触碰 Keychain/Credential Manager。
- `provider_save` 仅在传入非空 API Key 时写入系统凭据库。
- “已配置”只表示系统凭据存在，不代表已经成功完成一次模型请求。

## 7. 项目目录设置

### 7.1 数据模型

```json
{
  "schemaVersion": 1,
  "defaultProjectDirectory": "/Users/example/Documents/Sion Projects"
}
```

该文件保存于应用数据目录，不进项目目录。`defaultProjectDirectory` 可以为 `null`。

### 7.2 Tauri 命令

| 命令 | 用途 |
|---|---|
| `settings_get` | 读取默认项目目录。 |
| `settings_pick_default_project_directory` | 打开系统目录选择器并保存所选目录。 |
| `settings_clear_default_project_directory` | 清空默认目录。 |
| `project_create` | 使用默认目录作为目录选择器的初始位置，用户仍可改选。 |

设置写入使用原子写；当保存的目录后来不存在或不可访问时，目录选择器忽略该值并回退到系统默认位置，同时在设置页提示用户重新选择。

## 8. 删除历史迁移

因为确认没有需要保留的旧版用户数据，本次会移除：

- 启动页的“旧数据迁移”面板；
- `migration` 与 `provider_migration` Tauri 命令和实现；
- 迁移夹具、Golden 契约、迁移测试和 README 中的迁移说明；
- 仅为读取旧浏览器搜索设置而保留的兼容描述。

删除后，Sion 只创建和读取新的 `<用户选择的项目目录>/.sion/`。这是一项不可逆的产品范围收缩：旧网页版项目将不再能通过新桌面应用导入。

## 9. 数据、安全与运行时边界

| 数据 | 位置 | 规则 |
|---|---|---|
| 项目 Markdown、聊天、附件、Agent 运行记录 | `<项目根目录>/.sion/` | 原子写入；节点使用 revision/CAS。 |
| 最近项目、默认项目目录、提供商元数据 | 操作系统应用数据目录 | 不含 API Key。 |
| API Key | macOS Keychain / Windows Credential Manager | 不通过 IPC 回显，不写项目文件。 |
| 文件预览 | Rust 本地提取文本 | 不启动浏览器，不访问网络。 |

模型请求仍然只在用户发起 Agent 运行时发生；React UI 不直接访问文件系统或模型网络。

## 10. 实施阶段与验收

### 阶段 A：产品边界与设置

- 删除迁移入口和运行时代码。
- 增加设置存储及默认项目目录选择。
- 验收：未设置、已设置、目录失效、取消选择四种情况均不写入错误目录。

### 阶段 B：模型连接管理

- 增加连接详情、编辑、替换密钥、设默认和删除确认。
- 验收：编辑不丢失 Key；替换后旧 Key 不再存在；默认切换后 Agent 使用新提供商；界面从不显示密钥。

### 阶段 C：工作台结构

- 实现“节点 / 对话或交付稿 / 资料预览”三栏。
- 新增安全的文件提取文本预览命令与截断策略。
- 验收：选择资料不影响内容，勾选资料才改变 Agent 上下文；对话、交付稿、预览和 DOCX 导出均可用。

### 阶段 D：视觉系统与可读性

- 替换深色主题、建立设计 token、提高最小字号和控件尺寸。
- 验收：在 macOS Apple Silicon 上用键盘与鼠标完整走通新建、设置、模型编辑、对话、文件预览和导出；检查 1080px 及最小窗口宽度布局。

### 阶段 E：回归与发布

- 更新 README、AGENTS.md、无浏览器运行时检查与 CI。
- 执行 Rust 单元测试、前端 TypeScript 检查、DMG 打包和 Windows CI。

## 11. 评审结论要求

进入实现前，需确认以下结论：

1. 保留“每次新建都选择目录”，默认项目目录只控制系统选择器初始位置。
2. 接受删除旧项目迁移功能，旧网页版数据不再导入。
3. 接受中间区域以“对话”为默认页签，交付稿改为同区域页签。
4. 接受右栏首版只显示提取文本预览，不提供 PDF 页面渲染或浏览器预览。
5. 接受模型连接编辑时 API Key 只能替换，不能查看明文。
