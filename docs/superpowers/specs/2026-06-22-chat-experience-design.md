# Sion 对话体验升级设计

## 1. 背景

当前工作台已经支持流式对话、模型思考内容、联网检索、交付稿增量更新和 Markdown 预览，但用户难以快速判断 Agent 正在执行什么操作。Agent 回复仍以纯文本展示，思考框在折叠时缺少进行中的反馈，交付稿预览也只有基础排版。页面在部分桌面宽度下还会产生页面级横向滚动。

模型调用目前没有采集或持久化 token usage，因此无法展示单轮对话和当前会话的实际消耗。

本设计是两期改造中的第一期，只处理对话体验。文件池对 PDF、Word、Excel 的支持，以及会话保留与删除界面，归入后续“项目资产”设计。

## 2. 目标

- 让用户持续看见 Agent 当前阶段，避免把正常处理误认为卡顿。
- 将 Agent 回复渲染为易读、安全的 Markdown。
- 将交付稿预览升级为具有正式文档观感的阅读界面。
- 统计每一轮触发的全部模型调用，并提供当前会话累计 token 用量。
- 保证工作台在 1280px 及以上桌面视口不产生页面级横向滚动。
- 沿用本地优先架构，不引入数据库或外部分析服务。

## 3. 非目标

- 不在本期解析 PDF、DOC/DOCX、XLS/XLSX。
- 不在本期修改每个流程节点最多保留 10 个会话的策略。
- 不在本期增加会话删除入口。
- 不为平板或手机重新设计完整工作流。
- 不建立完整的运行事件审计账本。
- 不把本地估算 token 当作服务商账单值。

## 4. 设计方向

采用“专业文档工作台”视觉方向：克制配色、清晰层级、精致排版和轻量动效。实现采用渐进式升级，扩展现有 SSE 和本地 JSON 数据结构，不重写现有聊天与交付稿流程。

工作台仍保持节点、对话、交付稿三栏结构。组件职责调整如下：

- `ChatPanel` 负责会话编排、发送与 SSE 消费。
- 独立消息组件负责用户消息、Agent Markdown、来源和本轮 usage。
- 独立思考组件负责折叠内容、动态摘要和计时。
- 独立状态组件负责展示当前 Agent 阶段。
- 统一 Markdown 渲染组件负责解析、安全规则和通用元素映射。
- 聊天与交付稿分别提供紧凑内容和纸张文档两种视觉皮肤。
- project-domain 类型、usage 聚合和估算逻辑放在 `src/lib/project`。

## 5. Agent 状态模型

### 5.1 阶段

服务端通过结构化 SSE 事件表达阶段，前端不得通过中文提示文本反推状态。

| 阶段 | 含义 | 视觉 |
| --- | --- | --- |
| `idle` | 等待用户输入 | 中性灰 |
| `thinking` | 分析请求或模型推理 | 靛蓝呼吸点 |
| `reading_files` | 读取所选项目文件或 URL | 青色 |
| `searching_web` | 搜索或抓取网页 | 蓝色 |
| `generating_answer` | 生成可见回复 | 紫色流动指示 |
| `updating_document` | 判断或写入交付稿 | 琥珀色 |
| `completed` | 本轮成功完成 | 绿色 |
| `failed` | 本轮失败 | 红色 |
| `interrupted` | 用户主动停止或连接中断 | 中性警示色 |

阶段事件包含阶段、用户可读摘要和发生时间。未知阶段在客户端降级为“处理中”，但必须在本轮结束时回到终态。

### 5.2 展示行为

- 当前状态显示在聊天栏顶部，使用颜色圆点、阶段标签和简短说明。
- `completed` 短暂显示后回到 `idle`；`failed` 和 `interrupted` 保留到下一次操作或用户关闭。
- 联网搜索、URL 读取和交付稿状态整合进同一状态区，验证提示等需要用户操作的内容仍使用独立通知。
- 所有循环动画响应 `prefers-reduced-motion`。

## 6. 思考体验

思考框默认折叠。折叠时仍展示：

- 正在进行的动态标题；
- 从本轮开始计算的已耗时；
- 最新阶段摘要，例如“正在分析需求”或“正在整理检索结果”；
- 表示仍在工作的轻量呼吸动画。

用户展开后可以查看当前已收到的流式思考文本。回答开始后停止进行中动画，标题改为“已思考 N 秒”。历史消息只显示最终耗时和保存的思考内容，不重新播放动画。

如果服务商不提供思考文本，处理中仍显示阶段和耗时，但不生成虚假的思考内容。

## 7. 消息与 Markdown

### 7.1 Agent 回复

Agent 回复使用 Markdown 渲染，不再使用 `white-space: pre-wrap` 直接显示。支持 GFM 标题、列表、任务项、表格、引用、链接、行内代码、代码块和分隔线。

- Agent 内容采用轻量卡片，不使用厚重气泡。
- 代码块显示语言标签和复制操作。
- 宽代码块与宽表格仅在自身容器中横向滚动。
- 外部链接在新窗口打开并使用安全的 `rel` 属性。
- 不允许 Markdown 中的原始 HTML 直接注入页面。
- Markdown 渲染意外失败时降级显示纯文本。
- 流式新增内容采用轻微淡入和上移动效；减少动态效果时取消位移动画。

用户消息继续使用右侧紧凑气泡，保留换行，不强制按 Markdown 渲染。

### 7.2 交付稿预览

“预览交付稿”使用独立文档皮肤：

- 灰白工作区内放置居中的白色纸张；
- 限制舒适阅读宽度，提供清晰页边距与轻阴影；
- 为中文标题、正文、列表、表格、引用和代码建立独立排版；
- 顶部显示文档标题、字数、更新时间和复制操作；
- 流式生成期间继续更新预览内容，但不自动切换用户当前标签页。

“编辑 Markdown”继续提供原始内容编辑，用户可随时在编辑与预览间切换。

## 8. Token 用量

### 8.1 统计边界

用户每次发送消息时创建一个 `turnId`。本轮触发的全部模型调用都归入该轮，包括：

- 面向用户的回复；
- 工具调用规划和续轮；
- 联网检索相关模型调用；
- 事实判断；
- 当前对话触发的交付稿处理。

普通文件读取、浏览器搜索请求和本地计算本身不产生模型 token，不计入 usage。

### 8.2 数据结构

每次模型调用产生一条 usage 明细，至少记录：

- 调用类别；
- 输入 token；
- 输出 token；
- 总 token；
- `exact` 或 `estimated` 来源；
- provider 与 model；
- 调用完成状态。

Agent 消息保存本轮汇总 usage 和 `turnId`。会话累计值在读取会话时从消息中的有效 usage 聚合，不维护独立的可漂移计数器。

### 8.3 精确值与估算值

- 模型协议返回 usage 时优先使用精确值。
- 服务商未返回 usage 时，根据实际发送给模型的输入和已接收输出进行本地估算。
- 本地估算必须标记为 `estimated`，界面显示“估算”，不得表现为账单精确值。
- 同一轮同时包含精确和估算明细时，汇总标记为“含估算”。
- 旧历史消息没有 usage 时显示“暂无统计”，不对历史内容补算。
- 中断或失败时保存已经取得的有效 usage；无法可靠得到的数据不进入累计。

### 8.4 界面

- 每条 Agent 回复底部显示紧凑 token 标签。
- 点击标签展示本轮输入、输出、总量以及精确/估算标记。
- 会话标题旁提供“本会话用量”入口，展示消息中有效 usage 的累计值。
- 默认不展示内部调用明细；数据结构保留明细，以便未来扩展。

## 9. 数据流

1. 客户端提交消息，服务端解析或创建 `turnId`。
2. 服务端发送 `thinking` 状态并启动计时。
3. 读取文件、联网检索、生成回复和更新交付稿时发送对应阶段。
4. 每个模型适配器提取协议 usage；没有 usage 时调用统一估算器。
5. usage 收集器按 `turnId` 合并所有模型调用。
6. 服务端把回复、思考内容、来源、turn usage 和时间信息保存到 Agent 消息。
7. 最终 SSE 事件返回会话 ID、消息 ID、终态和汇总 usage。
8. 客户端用服务端最终消息替换临时流式消息，并更新会话累计展示。

服务端保存是 token 统计的事实来源。前端只负责流式临时展示，不自行永久累计。

## 10. 布局与滚动

- 目标视口为宽度 1280px 及以上的桌面环境。
- 三栏使用受约束的弹性宽度，任何一栏内容都不能撑大页面网格。
- 左侧节点栏提供折叠能力，为中间与右侧释放空间。
- 聊天消息、Markdown 容器和 flex/grid 子项必须允许收缩。
- 页面根容器禁止横向溢出；代码块与表格可在局部横向滚动。
- 本期不承诺平板或手机完整工作流。

## 11. 错误与兼容

- SSE 失败、用户停止或连接中断时立即停止动效并进入相应终态。
- 尽量保留已接收回复、思考、来源和有效 usage，不因最终事件缺失清空内容。
- usage 中的负数、非有限数或不一致总数视为无效，不参与聚合。
- 旧消息和会话 JSON 通过可选字段保持向后兼容，不要求一次性迁移。
- Markdown 渲染异常降级为纯文本；单条消息异常不得破坏整个会话。
- 未知 SSE 阶段降级显示“处理中”，结束事件必须清理进行中状态。

## 12. 测试策略

行为变更遵循测试先行。测试至少覆盖：

- 两种模型协议的 usage 提取；
- usage 估算、合并、校验和“含估算”语义；
- 全部内部模型调用归入同一 `turnId`；
- 历史消息缺少 usage 时的兼容；
- SSE 阶段顺序、未知阶段、中断和失败终态；
- 思考框默认折叠、动态摘要、计时与完成状态；
- Agent GFM 渲染、链接安全和原始 HTML 不执行；
- 代码块和表格的局部滚动约束；
- 单轮与会话 token 弹层聚合；
- 交付稿纸张样式及生成期间预览更新；
- 1280px 视口下无页面级横向滚动；
- 现有聊天、联网检索和交付稿生成回归。

完成实现后运行：

```bash
npm run test
npm run lint
npm run typecheck
```

## 13. 验收标准

- 用户在整个请求期间都能看到准确的 Agent 阶段、说明和运行反馈。
- 思考框默认折叠，但折叠状态仍能证明系统正在持续处理。
- Agent 回复按 Markdown 正确渲染，页面不会被宽内容撑出横向滚动。
- 交付稿预览具有独立、正式的文档阅读样式。
- 新生成的每条 Agent 回复可查看本轮总 token，当前会话可查看累计 token。
- 精确值、估算值和不可用状态在界面上明确区分。
- 每轮 token 包含该轮触发的全部模型调用。
- 1280px 及以上桌面视口没有页面级横向滚动。
- 旧项目与旧会话无需迁移即可继续打开和使用。
