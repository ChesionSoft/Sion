# Sion 操作手册

这份手册面向实际使用者，帮助你从空项目开始，完成模型配置、项目创建、节点推进、文件引用、浏览器搜索、Agent 规则调整和分阶段正式 PRD 导出。

## 1. 启动 Sion

在项目目录执行：

```bash
npm install
npm run dev
```

打开：

```text
http://localhost:3000
```

## 2. 配置模型提供商

进入主菜单后，先打开“模型配置”区域。

添加模型提供商时需要填写：

- **提供商名称**：用于你在界面中识别，例如 `OpenAI`、`DeepSeek`、`Qwen`。
- **API 协议**：选择服务商支持的 Chat Completions 或 OpenAI Responses。
- **地址模式**：选择“由系统补全路径”时填写 API Base URL；选择“完整 API URL”时填写服务商给出的完整端点。
- **API Key**：服务商提供的密钥。
- **模型列表**：填写可以调用的模型名称，例如 `gpt-4.1`、`deepseek-chat`。
- **工具**：仅为支持函数工具调用的模型勾选；未勾选时 Sion 使用规划器兜底。这个选项不决定联网搜索是否可用。
- **默认模型**：点击星标设置默认模型。
- **上下文长度**：可选，用来帮助你识别模型适合读取多长资料。

配置完成后，所有项目都可以使用这些模型。进入项目后，你仍然可以在聊天框或导出中心选择具体模型和推理强度。

选择“由系统补全路径”时，Chat Completions 会使用 `/v1/chat/completions`，OpenAI Responses 会使用 `/v1/responses`；选择“完整 API URL”时，Sion 不会改写你填写的地址。

主页的“浏览器搜索”区域可配置 Google / 百度与浏览器偏好。搜索由本地安全代理执行；可选择系统 Chrome/Edge 或托管 Chromium，不需要另行配置搜索 API Key。托管 Chromium 不会自动下载，需点击“安装托管 Chromium”后使用。

浏览器搜索设置不会自动开启聊天联网。进入一个节点会话后，点击输入框左侧的地球图标打开“联网搜索”；开关按会话保存。若用户消息直接包含 HTTP(S) 链接，Sion 也会自动尝试读取该网页内容。

## 3. 创建项目

在主菜单填写：

- **项目名称**：例如“库存管理系统”。
- **客户名称**：可选，用于交付文档。
- **编制方**：可选，用于 Word 文档作者信息。

点击“创建项目工作台”后进入项目详情页。

## 4. 理解工作台布局

项目工作台分为三栏：

- **左侧：12 节点设计路径**  
  用于切换当前设计阶段，例如项目基本信息、功能模块、接口设计、开发任务等。

- **中间：当前节点 Agent 对话**  
  用于和当前节点的 Agent 讨论需求、澄清边界、引用文件并生成可写入交付稿的内容。

- **右侧：节点交付稿**  
  用于编辑、预览和保存当前节点的 Markdown 内容，也可以查看或自定义该节点的 Agent 规则。

顶部工具区包含：

- **返回主菜单**：回到项目列表和模型配置。
- **项目文件池**：管理当前项目可引用的资料。
- **导出中心**：审阅正式 PRD 蓝图、正文、质检报告并下载通过质检的 Word。

## 5. 按节点推进文档

建议从第 1 个节点开始逐步推进：

1. 查看右侧默认 Markdown 结构。
2. 在中间聊天框补充已知信息。
3. 让 Agent 追问关键问题或整理章节内容。
4. 将确认后的内容写入右侧 Markdown。
5. 点击“保存当前节点交付稿”。
6. 切换到下一个节点继续。

每个节点都应尽量区分：

- 已确认内容
- 设计假设
- 待确认问题

这样最终导出文档会更清楚，也方便客户或团队后续确认。

## 6. 使用项目文件池

点击右上角“项目文件池”，上传项目资料。

推荐上传 Markdown 文件，例如：

- 需求说明
- 会议纪要
- 客户补充材料
- 旧系统说明
- 接口草稿

上传后，在聊天框左下角点击附件按钮，勾选本轮对话需要引用的文件。被选择的文件会作为上下文交给模型阅读。

如果文件较大，模型调用可能消耗较多 token，也可能超过模型上下文限制。遇到上下文过长时，建议拆分文件或只选择当前节点需要的资料。

## 7. 选择模型和推理强度

聊天框右下角会显示当前模型和推理强度。

你可以按节点选择不同策略：

- **低**：简单整理、改写、轻量问答。
- **中**：一般设计节点的默认选择。
- **高**：复杂功能、接口、数据结构和架构讨论。
- **超高**：需要更深入推理的任务拆分、风险分析或复杂方案比较。

如果你配置了多个模型提供商，也可以在同一个选择器里切换模型。

## 8. 自定义单个节点的 Agent 规则

右侧切换到“Agent 规则”标签。

默认模式是“使用内置默认规则”，适合大多数项目，内容只读。

如果当前项目对某个节点有特殊要求：

1. 切换到“使用项目自定义规则”。
2. 从默认规则重新复制。
3. 修改规则内容。
4. 点击“保存项目 Agent 规则”。

自定义规则只影响当前项目的当前节点，不会修改内置默认规则。

## 9. 保存与分阶段导出

每个节点的 Markdown 都需要点击“保存当前节点交付稿”才会写入本地项目文件。

当主要节点内容完成后，从工作台进入“导出中心”，按顺序操作：

1. 选择模型，点击“生成导出蓝图”。蓝图决定哪些已确认内容适合对外，不会把待确认事项、过程讨论或 Agent 建议带进正式 PRD。
2. 审阅 `export-blueprint.md`，确认后点击“确认蓝图并生成正文”。
3. 审阅 `formal-prd-draft.md`，确认后点击“确认正文并生成正式 Word”。
4. Sion 在服务端生成 DOCX，并通过 LibreOffice/Poppler 渲染为 PDF/图片进行页面级质检。
5. 质检通过后，下载 `项目开发设计文档.docx`；质检失败时查看 `formal-prd-qa-report.md`，然后点击“重新生成正式正文”。

导出中心按需生成下列文件：

- `export-blueprint.md`
- `formal-prd-draft.md`
- `formal-prd-qa-report.md`
- `PROJECT_DESIGN.md`
- `项目开发设计文档.docx`
- `SPEC.md`
- `TASKS.md`
- `AGENTS.md`

这些文件位于当前项目的 `exports/` 目录下。

## 10. 本地数据与隐私

Sion 默认把项目数据保存在本机目录中：

```text
projects/
settings/
```

其中可能包含客户需求、API Key、聊天记录和导出文档。不要把这些目录提交到公开仓库。

## 11. 常见问题

### 为什么模型请求失败？

优先检查：

- API 协议、地址模式与服务商文档是否一致。
- Base URL 模式下是否只填写基础地址；完整 URL 模式下是否填写了完整端点。
- API Key 是否正确。
- 模型名称是否和服务商后台一致。
- 所选 API 协议是否为服务商和当前模型所支持。

### 为什么文件引用后提示上下文过长？

说明当前选择的文件内容加上项目上下文超过了模型可读取长度。可以减少文件数量、拆分 Markdown，或切换更大上下文模型。

### 为什么 Agent 没有直接修改右侧文档？

当前设计是“Agent 给建议，用户确认后写入交付稿”。这样可以避免模型误改已确认内容。你可以复制 Agent 输出中的建议内容，整理进右侧 Markdown 后保存。

### 自定义 Agent 规则会影响其他项目吗？

不会。项目自定义规则只保存在当前项目里。

### 为什么导出中心里的文件是灰色、无法选择？

导出中心会列出可能生成的交付物。灰色表示当前项目尚未生成该文件，或新一轮蓝图/正文已使旧 Word 与质检报告失效。完成对应步骤后，文件即可预览或下载。

### 为什么旧项目的 Word 预览或下载不可用？

新版正式 Word 必须有当前导出流程写入的质检通过状态。旧的一次性导出不含该状态；请重新走导出中心生成并质检新的正式 Word。
