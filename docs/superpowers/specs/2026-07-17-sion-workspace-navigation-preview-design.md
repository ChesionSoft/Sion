# Sion 工作台导航与交付稿预览重构设计

**日期：** 2026-07-17
**状态：** 对话设计已确认，等待书面规格审阅

## 背景

现有 Codex 风格工作台已经建立项目首页、持久侧栏、中央聊天区和可选右侧工作区，但仍保留了一套“已打开节点 + 可关闭节点 + 全部节点选择器”的临时分页模型。节点被选择后会在 `openedNodeIds` 中重排，用户无法把侧栏当作稳定的 12 步工作流目录。顶部同时放置资料、会话和运行记录下拉框，入口层级混杂；右侧交付稿默认进入 Markdown 编辑器，而不是更适合阅读的交付稿预览。

项目首页和导出中心目前还使用没有语义的方块字符，工作台其他入口也混用 Unicode 符号。图标无法稳定表达功能，且与参考的 Codex 单色线性图标风格不一致。

本设计把这些问题视为一次工作台信息架构重构，而不是七个互不相关的局部修改。现有 Tauri 数据边界、项目目录结构、Agent 调度、会话存储、文件提取、交付稿 CAS 保存和 DOCX 导出保持不变。

## 目标

- 打开项目后，在当前项目下固定展示全部 12 个工作流节点。
- 点击节点只切换当前上下文，不改变节点顺序，也不创建或关闭“已打开节点”。
- 将所有面向用户的“资料”文案统一改为“文件池”。
- 移除顶部会话日期下拉和运行记录下拉。
- 将聊天记录迁移到聊天区自身的工具栏和抽屉。
- 右侧默认显示当前节点的交付稿预览，并允许切换到同一份 Markdown 的源文件编辑视图。
- 将右上角操作稳定为“交付稿 / agent.md / 文件池 / 更多”。
- 为主导航和关键功能入口建立 Codex 风格的单色线性语义图标。
- 保持所有现有数据安全、脏稿保护和原生能力边界。

## 非目标

- 不改变 12 个节点的标识、顺序、领域含义或默认 Markdown。
- 不修改项目数据格式、项目目录布局或节点 CAS revision 规则。
- 不重写 Agent 调度、SSE 流式输出、取消运行、文件提取或 DOCX 导出。
- 不给每个具体项目分配随机颜色、首字图标、表情或可配置封面。
- 不给 12 个节点分别添加图标；节点目录保持纯文字。
- 不增加暗色主题、云同步、浏览器能力或新的导出格式。
- 不引入前端路由或大型状态管理库。

## 方案选择

采用“定向重构工作台信息架构”的方案：保留现有业务数据和 Tauri 命令边界，重构 `Sidebar`、`ProjectWorkspace`、右侧工作区和相关 UI 状态。

未选择以下两种方案：

1. **局部修补现有组件。** 仅修改文案、按钮和样式会继续保留“已打开节点”、可关闭分页和顶部下拉的旧模型，无法形成一致交互。
2. **完整重写前端工作台。** 重新实现所有状态管理会扩大对会话、脏稿保护、文件预览和持久化的回归风险，超出本次目标。

## 信息架构

### 左侧导航

主导航继续包含“项目”和“导出中心”。当前项目展开后固定渲染 `NODES` 定义的全部 12 个节点：

1. 项目基本信息
2. 需求背景与目标
3. 角色与权限
4. 业务流程
5. 功能模块
6. 页面与交互
7. 数据结构
8. 接口设计
9. 架构与部署
10. 开发任务
11. 风险与待确认
12. 最终文档

节点始终按以上顺序显示。选中态只通过字重和浅灰背景表达，不显示节点图标、关闭按钮或脏稿图标，不再提供“全部节点”按钮和节点选择器。脏稿风险由切换时的统一保护对话框处理。

项目根项可以折叠或展开；打开项目时默认展开。点击未激活节点时，先执行脏稿保护，再切换当前节点。点击当前节点不触发重新加载或重排。

### 中央聊天区

中央区域继续是当前节点的聊天工作区。顶部日期会话下拉被移除，聊天区增加一条轻量工具栏：

- 左侧“聊天记录”按钮打开当前节点的会话抽屉；
- 右侧“新会话”按钮创建并选择新会话；
- 抽屉列出现有会话名称、时间和消息数量，并允许切换会话。

抽屉只覆盖中央聊天区，不挤压左侧节点目录或右侧工作面。它不改变会话的本地存储格式和现有创建、选择、消息加载逻辑。

Agent 正在运行时，聊天区继续明确显示运行状态与停止操作。停止运行不能依赖“更多”菜单或运行记录加载成功。

### 顶部功能入口

项目工作台右上角稳定显示以下入口，顺序固定：

1. **交付稿**：文档/勾选语义图标和文字；打开右侧交付稿工作面。
2. **agent.md**：Agent/规则语义图标和文字；打开右侧 Agent 规则工作面。
3. **文件池**：文件夹/文件集合语义图标和文字；打开右侧文件池工作面。
4. **更多**：三点按钮；菜单中提供运行记录等低频操作。

顶部不再显示会话日期或运行次数下拉。正常宽度下三个核心入口同时显示图标和文字；紧凑宽度下可以只显示图标，但必须保留提示文字和无障碍名称。“更多”始终使用三点，不叠加额外图标或文字。

### 右侧工作面

右侧同时只显示一个工作面，不再以一排可关闭分页作为主要导航。支持以下工作面：

- 交付稿；
- agent.md；
- 文件池；
- 单个文件预览；
- Assistant 交付稿修改预览。

右侧面板仍可调整宽度和关闭。关闭后，顶部三个入口可以重新打开相应工作面。打开项目以及每次成功切换节点时，右侧都重置并打开该节点的交付稿预览。右侧宽度继续持久化。

文件池中的单个文件预览在同一右侧工作面中打开，并提供返回文件池的明确动作。Assistant 修改预览继续由消息中的预览操作打开；取消或应用后返回当前节点的交付稿预览。

## 交付稿预览与源文件

交付稿工作面顶部显示节点名称、状态和 revision，并提供“预览 / 源文件”分段切换：

- **预览**是默认视图，以排版后的 Markdown 阅读视图显示当前草稿；
- **源文件**显示 Markdown 编辑器、保存状态和保存操作；
- 两个视图共享 `draft`，不是两份文档；
- 未保存编辑切回预览时，应立即反映在预览中；
- 预览与源文件之间切换不触发脏稿保护；
- 保存继续调用现有 CAS 接口，revision 冲突时保留用户草稿。

预览明确支持标题、段落、列表、表格、代码块、行内强调和链接文本。原始 HTML 必须转义，不执行脚本；远程图片或其他外部资源不得自动加载。预览只负责本地排版，不为桌面运行时增加浏览器或网络访问能力。

导出 DOCX 和当前交付稿状态操作继续从交付稿工作面提供，不改变原生导出流程。

## agent.md 工作面

`agent.md` 需要展示三部分：

1. 当前节点的内置规则，只读；
2. 当前节点的自定义追加规则，可编辑；
3. Agent 最终收到的完整生效规则，只读。

React 不直接读取 `assets/agents/`。Tauri 命令层增加一个只读查询，按当前项目和节点返回：

```ts
type EffectiveAgentRules = {
  builtInMarkdown: string;
  customMarkdown: string | null;
  effectiveMarkdown: string;
};
```

该查询复用 Rust 中现有的内置规则和自定义覆盖读取逻辑。保存仍调用现有自定义规则接口，只允许写入或清除 `customMarkdown`；内置规则不能从 UI 修改。读取或保存失败只影响 agent.md 工作面，不阻塞聊天、交付稿或文件池。

## 文件池

所有用户可见的“资料”统一更名为“文件池”，包括：

- 顶部入口；
- 右侧工作面标题与说明；
- 空状态、导入按钮和错误提示；
- 文件不可用提示；
- 与 Agent 上下文选择相关的辅助文案。

文件池仍使用现有项目附件列表、原生导入、受限文本提取、Agent 上下文选择和单文件预览能力。存储目录、IPC DTO 和后端命名不因纯 UI 文案调整而强制迁移。

## 图标系统

新增一个小型、类型安全的内部 `Icon` 组件和受控 SVG 图标集合。图标遵循参考的 Codex 风格：

- 单色；
- 16–18 px 常用尺寸；
- 约 1.6 px 圆角线条；
- 不使用彩色底块、渐变、表情或抽象 Unicode 字符；
- 同一功能在侧栏、首页、导出中心和工作台中使用同一图标。

需要语义图标的入口包括：项目、导出中心、设置、项目根项、交付稿、agent.md、文件池、聊天记录和运行记录。12 个节点不使用图标。“更多”使用三点。

项目首页和导出中心中的项目项使用统一的“项目文档”线性图标；它表达条目的类型，而不是为每个具体项目制造不同身份。不同项目仍通过名称、当前节点和时间等文本识别。

## 组件边界

| 单元 | 职责 |
|---|---|
| `Sidebar` | 主导航、项目列表、当前项目固定 12 节点目录、折叠状态和设置入口。 |
| `ProjectWorkspace` | 工作台头部、聊天区与右侧工作面的组合，以及脏稿导航保护。 |
| `ConversationPane` | 消息、输入、流式响应、运行状态和停止操作。 |
| `ConversationHistoryDrawer` | 当前节点的会话列表、新建会话和会话切换。 |
| `RightWorkspacePane` | 单一右侧工作面、关闭、宽度调整和紧凑窗口覆盖行为。 |
| `DeliveryWorkspace` | 交付稿预览、Markdown 源文件、保存、revision 和导出入口。 |
| `AgentRulesWorkspace` | 内置、自定义和完整生效规则的展示与自定义规则编辑。 |
| `FilePoolWorkspace` | 文件列表、导入、上下文选择和文件预览导航。 |
| `DeliveryPreviewWorkspace` | Assistant 修改统计、完整预览、取消和应用。 |
| `Icon` | 受控的单色线性 SVG 图标集合。 |

`App.tsx` 继续协调 Tauri 命令、项目、节点、会话、消息、文件、运行和通知。展示组件不直接访问本地文件系统或模型网络。

## 状态与数据流

### 打开项目

1. 选择项目并进入工作台。
2. 从 UI 设置恢复 `activeNodeId`；不存在或无效时使用 `basic-info`。
3. 侧栏固定渲染全部 `NODES`，项目默认展开。
4. 并行读取当前节点、会话、文件、运行和 Agent 规则。
5. 右侧打开交付稿的预览视图。

### 切换节点

1. 若目标就是当前节点，不执行操作。
2. 若当前 Markdown 有未保存修改，显示现有“保存 / 放弃 / 取消”保护。
3. 确认切换后更新 `activeNodeId`，清除上一节点的瞬态请求结果。
4. 使用现有请求作用域防止迟到响应覆盖新节点。
5. 并行读取新节点、会话和 Agent 规则。
6. 右侧重置为交付稿预览，节点目录顺序不变。

### UI 设置兼容

应用设置 schema 继续保持版本 1。本次不要求迁移项目内容。

现有 `openedNodeIds`、`rightTabIds` 和 `activeRightTabId` 字段仍可被 Rust 读取，以保持旧设置文件兼容；新版前端不再用 `openedNodeIds` 决定节点目录，也不恢复任意右侧分页。保存 UI 设置时将节点列表规范化为固定 `NODES`，将默认右侧状态规范化为交付稿，并继续持久化 `activeNodeId`、侧栏折叠状态和右侧宽度。

Assistant 修改预览、单文件预览、聊天抽屉开关和“预览 / 源文件”切换都是瞬态状态，不跨应用重启恢复。

## 错误处理

- **节点读取失败：** 当前内容区显示错误与重试，不能继续显示上一节点内容。
- **会话读取失败：** 聊天记录抽屉显示错误；交付稿编辑保持可用。
- **Agent 规则读取失败：** agent.md 工作面显示错误；其他工作面保持可用。
- **文件导入或预览失败：** 错误留在文件池或文件预览中，不替换聊天内容。
- **运行记录失败：** “更多”菜单显示错误；活动运行的状态和停止操作仍由聊天区提供。
- **CAS 冲突：** 保留用户草稿并使用现有冲突处理界面，不覆盖磁盘新 revision。
- **迟到响应：** 所有节点、会话、消息和文件预览响应继续通过项目/节点/会话作用域校验。

## 可访问性与紧凑窗口

- 所有按钮具有可见键盘焦点和明确的无障碍名称。
- 图标不能成为唯一信息来源；正常宽度下核心入口同时显示图标和文字。
- 聊天记录抽屉支持键盘打开、关闭、遍历会话并把焦点返回触发按钮。
- 右侧工作面关闭后，焦点返回对应的顶部入口。
- 节点选中状态不只依赖颜色，同时使用字重和 `aria-current`。
- 侧栏可折叠；约 960 px 宽度下右侧工作面覆盖中央聊天区，而不是把三栏持续压窄。
- 紧凑宽度只显示功能图标时，每个按钮仍有 tooltip 和 `aria-label`。

## 验证

### 自动验证

前端纯状态测试覆盖：

- 12 个节点始终按 `NODES` 顺序呈现；
- 点击节点只更新 `activeNodeId`，不重排目录；
- 无效或旧 UI 设置安全回退；
- 打开项目和切换节点时右侧回到交付稿预览；
- 预览和源文件共享同一份草稿；
- 交付稿预览转义原始 HTML，且不会自动加载外部资源；
- 瞬态工作面不会写入持久 UI 设置；
- 旧 `openedNodeIds` 和右侧分页设置不会恢复旧交互。

Rust 测试覆盖完整 Agent 规则查询：

- 无自定义规则时，生效规则等于内置规则；
- 有自定义规则时，生效规则按现有运行时顺序组合；
- 查询不写项目数据；
- 保存自定义规则后查询返回更新内容；
- 内置规则不能通过该接口修改。

运行以下项目检查：

```bash
npm run lint
npm run build
npm run test:ui
npm run test:rust
```

### 桌面视觉验证

- 项目首页和导出中心使用明确的项目线性图标，不再显示方块字符；
- 打开项目后 12 个节点全部可见、纯文字、顺序固定；
- 选择不同节点不会移动条目；
- 聊天记录抽屉不会挤压左右区域；
- 顶部入口严格为交付稿、agent.md、文件池和更多；
- 运行记录位于更多菜单，活动运行仍可停止；
- 交付稿默认预览，源文件编辑后切回预览立即更新；
- agent.md 正确区分只读和可编辑区域；
- 约 960 px 窗口下仍可完成节点切换、聊天和交付稿编辑。

## 验收标准

- 打开项目后当前项目默认展开全部 12 个节点。
- 节点固定排序；点击节点只切换上下文，不置顶、不移到底部、不出现关闭操作。
- 12 个节点不显示图标。
- 项目首页、导出中心和关键功能入口使用统一的 Codex 风格语义线性图标。
- 所有用户可见的“资料”改为“文件池”。
- 顶部会话日期下拉和运行记录下拉被移除。
- 聊天记录从聊天区工具栏打开抽屉。
- 运行记录从“更多”菜单打开，活动运行状态和停止操作仍在聊天区可见。
- 右侧默认打开当前节点交付稿预览，并能切换到同一份 Markdown 的源文件编辑视图。
- 顶部三个核心入口都有图标和文字，顺序为交付稿、agent.md、文件池；其后是三点“更多”。
- agent.md 展示内置、自定义和完整生效规则，只允许编辑自定义部分。
- 现有项目数据、CAS 保存、Agent 运行、文件提取和 DOCX 导出行为不回归。
